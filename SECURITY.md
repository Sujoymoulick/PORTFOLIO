# Security Policy

## Supported Versions

Only the latest version on the `main` branch is actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

I take the security of this project and any associated services seriously. If you discover a vulnerability or security risk, please do not open a public issue. Instead, report it privately.

To report a vulnerability:
1. Send an email directly to **[sujoymoulick05@gmail.com](mailto:sujoymoulick05@gmail.com)**.
2. Include a detailed description of the vulnerability, steps to reproduce, and any potential exploits or proof of concepts (PoC).

### Response & Resolution Process

- **Acknowledgement**: I will acknowledge receipt of your vulnerability report within **48 hours**.
- **Investigation**: I will investigate the report and determine the impact.
- **Fix & Disclosure**: If verified, a security fix will be committed directly to the `main` branch. I will coordinate disclosure with the reporter to ensure safety.

Thank you for helping keep this project secure!
